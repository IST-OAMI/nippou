# 要件定義

## 概要

フォーマットに沿った日報を自動生成するPythonプロジェクト。

## 使い方

```bash
# 始業時: 5分間隔でスクショ+分析を繰り返す
/loop 5m /screenshot-log

# 終業時: 日報を生成
/nippou
```

## スキル仕様

### `/screenshot-log`（定期実行）

1. `screencapture` でスクリーンショットを撮影
2. `osascript` でアクティブウィンドウのアプリ名・タイトルを取得
3. Claude Code が画像+タイトルを分析し、作業内容をテキスト化
4. `logs/YYYY-MM-DD.md` にタイムスタンプ付きで追記
5. スクリーンショットファイルを削除

### `/nippou`（手動実行）

1. `logs/YYYY-MM-DD.md` を読み込み
2. `category.md` のカテゴリに基づき作業を分類
3. プロジェクト（`NOK_インフラ支援` / `IST_インフラ対応`）をログ内容から推測
4. 作業時間を 0.5h 単位で算出
5. `template.md` のフォーマットに沿って以下を自動生成:
   - 【今日の作業実績】: カテゴリ分類 + 時間
   - ＜やったこと＞: 作業詳細
   - 【学んだこと】: ログから学びを抽象化
6. `output/YYYY-MM-DD.md` に出力

## 情報ソース

| ソース | 取得方法 | 用途 |
|--------|----------|------|
| スクリーンショット | `screencapture` | 画面内容の分析 |
| アクティブウィンドウ名 | `osascript` | アプリ・ファイル名の特定、プロジェクト推測の補助 |

## 制約事項

- 作業時間は 0.5h 単位（0.5h, 1.0h, 1.5h...）
- 合計は 8.0h + 休憩 1.0h
- 手動記入: 所感、明日活かす具体策、人間関係
